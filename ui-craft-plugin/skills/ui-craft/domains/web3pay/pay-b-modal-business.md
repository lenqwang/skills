## 一、反馈弹窗（说明 / 引导 / 轻确认）

**业务含义**：向商户解释「将要发生什么」「为什么」「有什么好处」，或在做关键操作前做**轻量确认**；**不以复杂表单为主**。

**内容结构（从画布实例归纳）**

| 区域 | 常见内容 | 说明 |
|------|----------|------|
| **Header** | 短标题 | 多用 `ModalV5/Header-32px`（约 74px 高）；部分变体 Header 更高以容纳多行标题区 |
| **Body** | 一段主说明文案 | 例如开启某能力前的利益说明（英文化案示例：Auto Earn 相关说明） |
| **Body 延伸** | 要点列表（带勾选/标签样式） | 多条短句：安全、灵活性、计息方式等 **bullet 式** 信息 |
| **Footer** | `Modal Footer ButtonV5` | 主操作 + 取消/关闭；或单主按钮 |

**尺寸与档位（画布中出现的）**

- 约 **480px** 宽（V5 **Regular**）：短文案 + 双按钮 Footer（含 48px 高按钮的变体）。
- 约 **487–588px** 宽：说明 + 多条 Tag/要点 + 标准 Footer（**Regular ~ Medium**）。

**业务规范小结**

1. **文案**：先一句话说清「动作 + 结果」，下面用列表降低阅读成本。  
2. **层次**：主说明与要点列表区分字号/行高；避免大段无结构文字。  
3. **操作**：默认提供「继续/确认」与「取消」；危险操作若在此类弹窗出现，需在 PRD 中单独立项（是否升级为二次确认 / Critical 按钮）。  
4. **与全局反馈区分**：若仅为操作结果 toast/全局 Message 即可，不必占用 Modal；**需要展开说明或多项并列**时用本类。

## 二、配置弹窗（表单 / 规则 / 白名单等）

**业务含义**：商户**录入或修改配置**；多字段、可滚动、需明确提交/取消闭环。

**内容结构（从画布实例归纳）**

| 区域 | 常见内容 | 说明 |
|------|----------|------|
| **Header** | 配置标题 | 如画布文案：**「审批流规则配置」「IP地址白名单配置」** 等 |
| **Body** | 纵向表单 | 多个 `Input_V5`：Label + 输入区 +（可选）Helper；含 **password** 等特殊类型 |
| **Body** | 下拉/选择 | 如 `Select_V5` 浮层与表单项组合 |
| **Footer** | `Modal Footer ButtonV5`（约 88px 高） | 保存 / 取消 或 提交 / 返回 |

**尺寸与档位**

- 宽度多为 **588px**（V5 **Medium**）。  
- 高度随字段数量变化（画布中可见 **754 ~ 1100+ px**），与 [pay-b-modal.md](components/pay-b-modal.md) 中「最大高度 760px、内容区滚动」一致实现即可。

**业务规范小结**

1. **字段分组**：复杂配置按语义分组（可用标题或分段间距），避免单屏堆叠无分组。  
2. **校验**：必填、格式、权限错误优先 **行内 Helper** 或字段下提示；提交失败可用 Alert 条或顶部 `GTAlertV5`（画布中有预留实例，可按场景显示）。  
3. **敏感输入**：密码、密钥类使用专用输入组件，与明文字段区分。  
4. **提交结果**：成功后关闭弹窗并刷新列表/详情；失败保留弹窗与已填数据（除非安全要求清空）。

## 三、列表弹窗（明细 / 流水 / 带汇总的表格）

**业务含义**：在**不离开当前页**的前提下，展示**一组结构化记录**；常带**顶部汇总**与**可滚动表格**。

**内容结构（从画布实例归纳）**

| 区域 | 常见内容 | 说明 |
|------|----------|------|
| **Header** | 弹窗标题 | 如画布场景：**收益明细** 类（侧注「收益明细👉」指向本弹窗） |
| **Body 顶区** | 指标摘要 | 如 **「累计收益」+ 大号数字币种展示**（与下方表格联动） |
| **Body 主区** | `Modal FilterV5-web` + 表格 | 内嵌 `Table Header_V5` / `Table Cell_V5`；数据区 **纵向滚动**，可出现 **Scrollbar** |
| **可选** | 顶部告警 | `GTAlertV5` 在画布中为 **hidden** 占位 → 可按业务打开（筛选无效、数据延迟等） |
| **Footer** | 可无或仅「关闭」 | 画布 Large 实例以内容为主；若需导出/刷新，在 PRD 中单独定义 Footer 按钮 |

**尺寸与档位**

- **840 × 760**（V5 **Large** + 规范最大高度）：大表格 + 汇总条。  
- 遮罩 + 居中，与页面表格（如「理财币种」「统计时间」筛选）形成「主列表 → 行操作/入口 → 弹窗内子表」关系。

**业务规范小结**

1. **信息架构**：先 **KPI/汇总**，再 **明细表**，避免用户进弹窗先看到空表。  
2. **筛选**：若弹窗内仍需筛选，与 [pay-b-filter.md](components/pay-b-filter.md) 对齐；时间范围等放在表格上方。  
3. **行操作**：表内操作列（如画布中 `CEX_OrderDetails`）与主列表一致：**图标 + 热区**。  
4. **性能**：数据量大时默认分页或虚拟滚动（PRD 需写明）；滚动条仅在内容区，不把 Header 滚走。

## 四、三类对照总表

| 维度 | 反馈弹窗 | 配置弹窗 | 列表弹窗 |
|------|----------|----------|----------|
| **核心目的** | 说明 / 引导 / 轻确认 | 提交配置 | 展示批量明细 + 可选汇总 |
| **主体内容** | 短文 + 要点列表 | 多字段表单 | 汇总 + 表格 |
| **常见宽度** | 480 ~ 588px | 588px | 840px |
| **Footer** | 几乎必有 | 几乎必有 | 可有可无 |
| **滚动** | 少见 | 表单区长滚动 | 表格区滚动为主 |
| **pay-b-modal 映射** | Confirm / Detail（偏说明） | Form | Form（Large）或 Detail（若只读表） |
