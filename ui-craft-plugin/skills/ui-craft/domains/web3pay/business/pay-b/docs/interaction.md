# Pay-B 交互规范与可访问性

> 来源: Gate V5.1 设计系统 · Pay B端覆盖

---

## 交互规范

### 动画时长

| 场景 | 时长 | 缓动 |
|------|------|------|
| 微交互（Hover 反馈） | 100–150ms | ease-out |
| 标准交互（按钮点击） | 200ms | ease-in-out |
| 展开/收起 | 300ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 页面过渡 | 400ms | ease |
| 弹窗打开 | 250ms | ease-out |
| 弹窗关闭 | 200ms | ease-in |
| Toast 出现 | 300ms | ease-out |
| Toast 消失 | 300ms | ease-in，3 秒后自动 |
| Switch 切换 | 150ms | ease-out |

### 悬停效果

```css
/* 按钮 */
.btn:hover { box-shadow: 0 2px 8px rgba(0,0,0,0.15); }

/* 卡片 */
.card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 16px rgba(0,0,0,0.15);
}

/* 表格行 */
.table-row:hover { background: #FAFBFC; }
```

### 点击反馈

```css
.interactive:active { transform: scale(0.98); }
```

### 表单校验

| 触发时机 | 说明 |
|---------|------|
| 失焦校验 | 字段 blur 时立即校验 |
| 提交校验 | 点击提交时全量校验 |
| 实时校验 | 输入变化时（数量范围等） |

- 错误提示：字段下方红色文字 `#EF4444`，12px
- Placeholder：`请输入...` / `请选择...`

### DO / DON'T 反模式指引

> 以下列举常见设计反模式，供开发和设计走查对照。

#### 色彩

| DO | DON'T |
|-------|---------|
| 使用 CSS 变量引用色值（`var(--color-text-primary)`） | 直接硬编码 hex（`color: #070808`） |
| 用语义色区分状态：绿色=成功、红色=错误、橙色=警告 | 用蓝色同时表示"链接"和"成功"，混淆语义 |
| 不可逆操作使用红色危险按钮 | 删除按钮使用绿色主色 |

#### 按钮

| DO | DON'T |
|-------|---------|
| 一个页面/弹窗中只保留 1 个 Primary CTA | 同一视图中出现多个绿色主按钮 |
| 按钮文案使用"动词 + 宾语"（如"新建规则"） | 按钮文案仅用"确定"、"提交"等模糊词 |
| Loading 时禁用按钮防止重复提交 | 允许连续点击触发多次请求 |

#### 表单

| DO | DON'T |
|-------|---------|
| 失焦时实时校验并显示错误原因 | 仅在提交时才校验，且只提示"输入有误" |
| 必填字段用 `*` 标注 | 所有字段都不区分必填/选填 |
| 错误状态同时使用红色边框 + 图标 + 文字 | 仅靠红色边框传达错误（色觉障碍不可见） |

#### 弹窗

| DO | DON'T |
|-------|---------|
| 弹窗标题明确说明操作内容 | 标题写"提示" |
| 点击遮罩可关闭普通弹窗 | 所有弹窗都禁止遮罩关闭 |
| 安全验证弹窗中密码字段使用 `type="password"` | 安全验证明文显示资金密码 |

#### 布局

| DO | DON'T |
|-------|---------|
| 间距使用 `space-*` Token | 使用 `margin: 13px` 等非网格值 |
| 长文本使用 `text-overflow: ellipsis` 截断 | 长文本撑破容器或换行导致错位 |
| 所有 z-index 归入层级体系（0 / 10 / 100 / 200 / ... / 600） | 随意使用 `z-index: 99999` |

#### 图标

| DO | DON'T |
|-------|---------|
| 统一使用 Filled 图标风格 | 同一页面混用 Filled 和 Outlined |
| 图标颜色跟随文字颜色 Token | 图标使用硬编码颜色而文字使用 Token |

## 无障碍 / 可访问性（Accessibility）

### 色彩对比度（WCAG 2.1 AA）

| 场景 | 前景色 | 背景色 | 对比度 | 是否达标 |
|------|--------|--------|--------|---------|
| 主文字 / 白色背景 | `#070808` | `#FFFFFF` | 19.5:1 | AA / AAA |
| 二级文字 / 白色背景 | `#84888C` | `#FFFFFF` | 4.5:1 | AA (正文) |
| 三级文字 / 白色背景 | `#A0A3A7` | `#FFFFFF` | 3.1:1 | 仅限大字号 (>=18px Bold) |
| 禁用文字 / 白色背景 | `#C4C7CA` | `#FFFFFF` | 1.9:1 | 不达标（禁用态无需达标） |
| 主按钮白字 / 绿色背景 | `#FFFFFF` | `#68AD00` | 3.5:1 | 大字号达标；正文建议使用 `#FFFFFF` + 600 字重 |
| 链接蓝 / 白色背景 | `#0068FF` | `#FFFFFF` | 5.1:1 | AA |
| 错误红 / 白色背景 | `#EF4444` | `#FFFFFF` | 4.6:1 | AA (正文) |
| 成功文字 / 浅绿背景 | `#68AD00` | `#E6F4D2` | 3.2:1 | 配合图标使用 |

> **最低要求**：正文文字（< 18px）对比度 >= 4.5:1；大字号文字（>= 18px Bold 或 >= 24px Regular）对比度 >= 3:1。
>
> **三级文字 `#A0A3A7` 注意**：仅用于辅助说明、Placeholder 等非关键信息；关键信息至少使用 `#84888C`。

### 焦点状态（Focus State）

所有可交互元素必须具备可见的焦点指示器：

```css
/* 全局焦点样式 */
:focus-visible {
  outline: 2px solid #68AD00;
  outline-offset: 2px;
}

/* 深色背景上使用白色焦点环 */
.dark-bg :focus-visible {
  outline-color: #FFFFFF;
}

/* 按钮焦点态（补充阴影增强可见性） */
.btn:focus-visible {
  outline: 2px solid #68AD00;
  outline-offset: 2px;
  box-shadow: 0 0 0 4px rgba(104, 173, 0, 0.2);
}
```

| 组件 | 焦点表现 | Tab 顺序 |
|------|---------|---------|
| 按钮 | 绿色描边 `2px` + 外发光 | 按 DOM 顺序 |
| 输入框 | 边框变为 `#68AD00` | 表单内按顺序 |
| 链接 | 下划线 + 绿色描边 | 按内容流顺序 |
| 弹窗 | 焦点陷阱（Focus Trap），关闭后焦点归还触发元素 | 弹窗内循环 |
| 下拉菜单 | 箭头键上下选择，Enter 确认，Esc 关闭 | 面板内循环 |
| 表格行 | 无焦点（操作列按钮可聚焦） | 操作列按钮 |

### 色觉无障碍

- **不单独依赖颜色传达信息**：所有使用颜色区分的状态必须同时配备图标或文字标签
- **状态徽章**：`启用` / `停用` / `失败` 除了颜色差异外，还通过文字内容区分
- **表单错误**：红色边框 + 图标 + 错误文字，三重信号
- **图表**：如涉及折线/柱状图，使用不同线型（实线/虚线/点线）辅助区分

### 键盘导航

| 键位 | 行为 |
|------|------|
| `Tab` | 在可交互元素间正向移动 |
| `Shift + Tab` | 反向移动 |
| `Enter / Space` | 激活当前焦点元素（按钮、链接、开关） |
| `Esc` | 关闭弹窗 / 下拉 / 取消当前操作 |
| `↑ ↓` | 下拉菜单 / 选择器中上下选择 |
| `Home / End` | 列表中跳到首项 / 末项 |

### ARIA 标注建议

| 组件 | ARIA 属性 |
|------|----------|
| Modal | `role="dialog"`, `aria-modal="true"`, `aria-labelledby="modal-title"` |
| Toast | `role="alert"`, `aria-live="polite"` |
| Sidebar 菜单 | `role="navigation"`, `aria-label="主导航"` |
| 展开/收起 | `aria-expanded="true/false"` |
| 加载中 | `aria-busy="true"`, `aria-live="polite"` |
| 禁用按钮 | `aria-disabled="true"`（保留在 Tab 序列中） |
| 表格 | `role="table"`, 表头 `role="columnheader"` |
| 分页 | `role="navigation"`, `aria-label="分页"`, 当前页 `aria-current="page"` |

## Banner 规范

### 首焦 Banner — 类型 1（纯图文）

用于首页主 Banner，左侧文字 + 右侧图片布局。

| 元素 | 规格 |
|------|------|
| 大标题字号 | 56px |
| 大标题字重 | 600 Semibold |
| 副标题字号 | 20px |
| 副标题字重 | 500 Medium |
| 大标题与副标题间距 | 16px |
| 右侧图片固定尺寸 | 435 × 280px |
| 图片占栅格数 | 9 列（共 12 列） |
| 内容区最小间距 | 51px |
| 对齐方式 | 居中对齐 |
| 整体内容区域 | 1200px（居于页面中央） |
| Banner 区总高度（含 Header） | 440px |

**按钮规格：**
- 最多 2 个 Button，宽度固定 160px，高度 56px
- 按钮间距：16px

### 首焦 Banner — 类型 2（带卡片）

右侧为信息卡片而非纯图片时的规格。

| 元素 | 规格 |
|------|------|
| 右侧卡片宽度 | 486px |
| 卡片内 Main Text 字号 | 24px |
| 卡片内 Description 字号 | 16px |
| 图片比例 | 1:1 或 16:9 |
| 图片占位尺寸（1:1） | 96 × 96px |
| Main Text 最大行数 | 2 行（超出省略） |
| Description 最大行数 | 2 行（超出省略） |

### Banner 数据展示规范

用于 Banner 卡片内展示数字数据（如资产总额、收益率等）。

| 元素 | 字号 | 字重 |
|------|------|------|
| 大数字 | 28px | 600 Semibold |
| 数字说明文字 | 14px | 400 Regular |
| 数字与说明间距 | 4px | — |

### Banner 交互规范

| 行为 | 规则 |
|------|------|
| 翻页按钮显示时机 | 默认隐藏，鼠标 hover 时显示左右翻页图标 |
| Progress Bar 选中宽度 | 固定 12px |
| 文案截断规则 | 超出优先优化文案，仍超出则省略号截断 |
| 左右翻页图标尺寸 | 36 × 36px（含内 16px icon） |
