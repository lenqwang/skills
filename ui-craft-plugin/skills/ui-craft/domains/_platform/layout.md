# 第 4 层 — 页面布局

> 页面级骨架定义：模块组合顺序、栅格系统、响应式策略。
> style 文件的区块 5 只需声明 `layout: <类型>` 或 `layout: <类型>.<子页面>` 并列出差异项。

---

## 页面类型

### campaign（活动页）

> 适用于：空投、任务奖励、竞赛、VIP 活动、节日活动等营销页面。

#### campaign.home — 活动首页（默认）

```
Hero（标题 + 副标题 + CTA + 时间/倒计时）
    |  80px 上下内边距
友情提示（可选）
    |  var(--space-section)
活动区块 x N
    +-- 区块标题 + 描述
    +-- 奖池/统计信息
    +-- 表格/卡片/排行榜
    |  var(--space-section)
加码奖励（可选）
    |  var(--space-section)
相关产品推荐（可选）
    |  var(--space-section)
活动规则
    |  96px 底部间距
底部 CTA（可选）
```

#### campaign.detail — 活动详情页

```
面包屑导航（首页 > 活动中心 > 当前活动）
    |  24px
活动标题区
    +-- 标题（headline.large）
    +-- 状态标签（进行中/已结束）+ 时间范围
    |  var(--space-section)
活动内容区（单列）
    +-- 富文本 / 规则说明 / 奖品详情
    |  var(--space-section)
相关推荐（可选）
    +-- 相关活动卡片（3 列网格）
    |  var(--space-section)
活动规则（折叠面板）
    |  96px 底部间距
底部 CTA（可选，如"返回活动列表"）
```

## 栅格系统

### 默认栅格

| 属性 | 桌面 (>=1248px) | 平板 (768-1247px) | 手机 (<768px) |
|------|----------------|-------------------|---------------|
| 最大宽度 | 1200px | 100% | 100% |
| 列间距 | 24px | 24px | 16px |
| 行间距 | 32px | 32px | 24px |
| 页面内边距 | auto（居中） | 24px | 20px |

### 常用列配置

| 场景 | 桌面 | 平板 | 手机 |
|------|------|------|------|
| 全宽表格 | 1 列 | 1 列 | 1 列 |
| 功能卡片 | 3-4 列 | 2 列 | 1 列 |
| 排行榜 + 侧边 | 主区 70% + 侧栏 30% | 1 列 | 1 列 |
| 双栏对比 | 2 列 | 2 列 | 1 列堆叠 |

## 响应式策略

### 断点定义

```css
/* 手机优先 */
@media (min-width: 768px) { /* 平板 */ }
@media (min-width: 1248px) { /* 桌面 */ }
```

### 默认响应行为

| 元素 | 桌面 | 平板 | 手机 |
|------|------|------|------|
| Hero | 双栏（文案 60% + 插图 40%） | 单列堆叠 | 单列堆叠 |
| 活动区块 | 单列全宽 | 单列全宽 | 单列全宽 |
| 卡片列表 | 3 列 | 2 列 | 1 列 |
| 表格 | 全宽 | 全宽 | 横向滚动 |
| CTA 按钮 | 固定宽度 | 固定宽度 | 全宽 |
| 标题字号 | 100% | 85% | 70% |
| 面包屑 | 完整路径 | 完整路径 | 省略中间层级 |

### CSS 实现模式

```css
/* 容器 */
.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 20px;
}

@media (min-width: 768px) {
  .container { padding: 0 24px; }
}

/* 栅格 */
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 24px;
  }
}

@media (min-width: 1248px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

## style 文件引用方式

### 单页面（最常见）

style 文件的区块 5 只需声明页面类型 + 差异项：

```markdown
## 区块 5: 布局结构

> layout: campaign

### 差异项

- Hero: 数据大屏模式（4 列统计卡 + 排行榜预览），不使用默认双栏
- 活动区块: 包含 Tab 切换（赛季/周期）
```

如果完全使用默认布局，直接写：

```markdown
> layout: campaign | 同默认
```

### 多页面

当一个 style 涉及多个页面时，声明所有子页面：

```markdown
## 区块 5: 布局结构

> layout: portal.index + portal.detail

### 差异项

- portal.index: 内容最大宽度 ~1088px，纯单列，无多列卡片网格
- portal.detail: 增加安装命令栏 + Markdown 文档卡
```

### 引用规则

- `layout: campaign` — 等同于 `campaign.home`，单页面活动
- `layout: campaign.home + campaign.detail` — 多页面活动
- `layout: portal` — 等同于 `portal.index`，单页面产品页
- `layout: portal.index + portal.detail` — 多页面产品页
- 省略子页面名时默认使用第一个子页面（.home / .index）
