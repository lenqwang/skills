# Loading 加载

> 包含 5 种加载类型（骨架屏/全局/局部/底部/数字滚动），按使用场景分级，覆盖从首次加载到数值变动的全场景。

---

## Figma 链接

- [Loading_Web Guidelines](https://www.figma.com/design/kQIx9xKLiyO6KZIbazNSM2/?node-id=8491-5402)

---

## 类型总览

| 类型 | 使用场景 | 优先级 |
|------|---------|-------|
| 骨架屏 Skeleton | 页面首次加载、数据量大的一级/重要页面 | 最高 |
| 全局加载 Page Loading | 整页数据一次性请求 | 次高 |
| 局部加载 Section Loading | 页内大型模块（K 线图、评论区等） | 中 |
| 底部加载 Lazy Load | 列表懒加载（聚合搜索等） | 中 |
| 数字滚动 Number Scroll | 数值变化展示（资产、币价等） | 按场景 |

---

## 骨架屏（Skeleton Screen）

用于让用户在等待加载时预览页面基础架构。

### 视觉规格

| 属性 | 数值 |
|------|------|
| 色块圆角 | **20 px** |
| 扫光倾斜角度 | **-10 度** |
| 扫光宽度 | 约占版心宽的 **40%**（1920 px 屏宽时约 480 px） |
| 日间扫光渐变 | `#FFFFFF`（0% -> 100%）`Layer/3` |
| 日间色块颜色 | `#FAFAFA`（`Bg/Secondary`） |
| 夜间扫光渐变 | `#1F2023`（0% -> 100%）`Layer/3` |
| 夜间色块颜色 | `#131516`（`Bg/Secondary`） |

### 设计规则

**DO：**
- 使用大面积色块概括内容，不要逐字段精细还原
- 相近元素合并为一个色块
- 单独占行的图标或文字，尽可能与相邻内容合并
- 需为所有一级页面、重要页面或流程搭配骨架屏

**DON'T：**
- 不要将骨架屏元素过于精细（如每行文字单独一条），否则增加视觉噪音

## 全局加载（Page Loading）

用于整页数据一次性请求场景。

- 优先展示全局加载样式，**不展示空状态**
- **不应与其他 Loading 样式同时出现**
- Spinner 尺寸：**32 x 32 px**，居中显示于页面中央
- 支持 Light / Dark 双主题

---

## 局部加载（Section Loading）

用于页内大型模块（K 线图、评论区、文章内容等）。

- 加载区域占当前页面较大部分
- 优先展示局部加载样式，**不展示空状态**
- **不应与其他 Loading 样式同时出现**
- Spinner 尺寸：**24 x 24 px**，居中于加载区域

---

## 底部加载（Lazy Load）

适用于列表懒加载场景（聚合搜索页、行情列表等）。

- 列表滚动到底部时触发，显示底部 Loading 状态
- Spinner 尺寸：**16 x 16 px**（`Circular_Loading_无渐变_黑`），显示于列表底部
- 按钮内 inline loading 同样使用 16 px

---

## 数字滚动加载（Number Scroll）

仅适用于数值变动场景（个人资产总额、币价变动等）。

- 样式为数字槽位滚动动画
- 由设计主动判断是否使用，无固定必须应用场景
- 常见应用：首页资产面板、行情价格数字

---

## Spinner 尺寸速查

| 场景 | Spinner 尺寸 |
|------|-------------|
| 全局加载 | 32 x 32 px |
| 局部加载 | 24 x 24 px |
| 底部加载 / 按钮 inline | 16 x 16 px |
