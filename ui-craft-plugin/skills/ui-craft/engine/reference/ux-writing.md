# UX 文案指南

> 适用于 ui-craft 生成的所有页面。本文件是 R30（UX 文案质量）的原理支撑。
> 改编自 Impeccable frontend-design reference/ux-writing.md，适配 Gate.io 品牌调性。

---

## 核心原则

1. **具体优于模糊** — 用户不需要猜
2. **动作优于状态** — 告诉用户做什么，而非描述系统状态
3. **品牌调性一致** — 理性、专业、直接（见 design-contract.md）
4. **国际化友好** — 中文写完后考虑翻译到英文/多语言是否通顺

---

## 按场景分类

### 1. CTA 按钮

**规则：动词 + 目标对象**

| ❌ 避免 | ✅ 推荐 | 场景 |
|---------|---------|------|
| 确定 | 领取奖励 | 奖励领取弹窗 |
| 提交 | 立即参与 | 活动报名 |
| 取消 | 暂不参与 | 放弃操作（明确说明后果） |
| OK | 我知道了 | 纯信息确认 |
| 点击查看 | 查看排行榜 | 跳转链接 |
| 去交易 | 前往现货交易 | 交叉引流（说明去哪） |

**层级规则：**
- 主 CTA：正向动作，视觉最强（"立即参与"）
- 次 CTA：替代动作，视觉次强（"查看详情"）
- 取消/关闭：负向动作，最弱（文字链接或灰色按钮）

**禁止：**
- 两个按钮文案语义相近（"确认" vs "确定"）
- CTA 文案超过 6 个汉字（按钮过宽影响布局）

### 2. 错误信息

**公式：发生了什么 + 为什么 + 怎么解决**

| ❌ 避免 | ✅ 推荐 |
|---------|---------|
| 操作失败 | 领取失败：活动已结束。查看其他进行中的活动 → |
| 请求错误 | 网络连接异常，请检查网络后重试 |
| 未知错误 | 系统繁忙，请稍后重试。如持续出现请联系客服 |
| 参数错误 | 交易数量需大于 10 USDT |
| 403 | 当前地区暂不支持此活动 |

**规则：**
- 不暴露技术细节（HTTP 状态码、堆栈信息）
- 不使用幽默/俏皮语气（金融场景下用户遇到错误时情绪敏感）
- 如果用户可以自行解决 → 给出明确操作指引
- 如果用户无法解决 → 给出客服/帮助中心入口
- 错误信息中的操作链接用箭头（→）而非"点击这里"

### 3. 空状态

**原则：空状态是引导机会，不是死胡同**

| ❌ 避免 | ✅ 推荐 | 场景 |
|---------|---------|------|
| 暂无数据 | 还没有参与记录，立即参加活动赢取奖励 | 活动参与记录 |
| 无结果 | 未找到匹配的活动，试试其他筛选条件 | 搜索/筛选 |
| 列表为空 | 排行榜将在活动开始后更新 | 活动未开始 |
| — | 暂无奖励记录。完成任务即可获得奖励 → | 奖励列表 |

**结构：**
```
[插图/图标（可选）]
[说明文案（一句话，解释为什么是空的）]
[引导 CTA（可选，指向下一步操作）]
```

---

### 4. 加载状态

**规则：说明正在做什么，而非"加载中"**

| ❌ 避免 | ✅ 推荐 | 场景 |
|---------|---------|------|
| 加载中... | 正在获取排行榜数据... | 排行榜 |
| Loading | 正在提交参与信息... | 报名 |
| 请稍候 | 正在计算奖励金额... | 奖励计算 |

**规则：**
- 骨架屏场景不需要文案（视觉已表达"加载中"）
- 需要文案的场景：提交操作的等待、耗时较长（>2s）的数据请求
- 超过 5s 未完成：展示"时间较长，请耐心等待"或提供取消选项

---

### 5. 成功状态

**规则：确认结果 + 下一步**

| ❌ 避免 | ✅ 推荐 |
|---------|---------|
| 成功 | 报名成功！完成任务即可领取奖励 |
| 操作成功 | 奖励已发放至您的账户 → 查看资产 |
| ✓ | 已复制邀请链接，分享给好友赢取额外奖励 |

---

### 6. 确认对话框

**规则：标题说动作，正文说后果**

```
标题：确认退出活动？
正文：退出后当前进度将不会保留，已获得的奖励不受影响。
[取消]  [确认退出]
```

**禁止：**
- 标题用问号但正文不解释后果
- 两个按钮都是肯定语气（"确认" / "好的"）
- 破坏性操作的确认按钮用品牌色（应用警告色 `var(--error)` 或中性色）

---

### 7. 表单与输入

**标签规则：**
- 标签在输入框上方（非 placeholder 替代）
- placeholder 用于格式提示（"请输入邮箱地址"），不用于必填说明

**验证信息：**
| ❌ 避免 | ✅ 推荐 |
|---------|---------|
| 格式错误 | 请输入有效的邮箱地址，如 name@example.com |
| 必填 | 请输入交易数量 |
| 长度不够 | 密码至少需要 8 个字符 |

**实时验证 vs 提交验证：**
- 格式类（邮箱、手机号）→ 失焦时验证
- 业务类（余额不足、额度超限）→ 提交时验证
- 永远不要在用户输入过程中显示错误（至少等失焦）

---

### 8. 数值与金额

**规则：**
- 金额始终带币种单位：`1,000 USDT` 而非 `1000`
- 千分位分隔符：`1,234,567.89`
- 百分比保留合理精度：收益率 `12.5%`，不写 `12.5000%`
- "约"/"≈" 用于估算值，精确值不加

**禁止：**
- 金额无单位（用户不知道是 USDT 还是 BTC）
- 小数点后超过 2 位（除 BTC 等高精度币种外）

---

### 9. 时间与日期

**规则：**
- 活动时间用绝对时间 + 时区：`2026-03-15 18:00 (UTC+8)`
- 倒计时用相对时间：`距结束还有 2天 05:30:21`
- "刚刚" / "3 分钟前" 仅用于社交场景，活动页用绝对时间

## 语气规范

### Voice（品牌人格，始终一致）

基于 design-contract.md 的品牌调性：
- **理性** — 数据驱动，不做情感操控
- **专业** — 结构清晰，行为可预测
- **直接** — 结论先行，不兜圈子
- **全球化** — 中性语言，无文化偏见

### Tone（场景语调，随情境调整）

| 场景 | 语调 | 示例 |
|------|------|------|
| 活动宣传 | 积极但克制 | "参与交易赛，瓜分 100,000 USDT 奖池" |
| 错误/失败 | 平静、有帮助 | "网络异常，请检查连接后重试" |
| 成功/奖励 | 简洁确认 | "奖励已发放至您的账户" |
| 风险提示 | 严肃、明确 | "数字资产交易存在风险，请谨慎评估" |
| 规则说明 | 中性、精确 | "每位用户每日最多参与 3 次" |

## 国际化注意事项

- 中文文案长度通常是英文的 60-70%，UI 需为英文留出空间
- 避免依赖语序的文案拼接（`"恭喜" + name + "获得奖励"` → 用模板 `t('reward.success', { name })`）
- 避免使用中文特有的修辞（谐音梗、成语缩写）
- 数字格式区域敏感（千分位: 英文 `,` 中文 `,`，但德语用 `.`）

## 检查清单（对应 R30）

- [ ] CTA 按钮是否为"动词 + 目标"格式？
- [ ] 是否存在"确定/取消/提交/OK"等模糊文案？
- [ ] 错误信息是否包含原因和解决方案？
- [ ] 空状态是否提供了下一步引导？
- [ ] 加载文案是否说明了正在做什么？
- [ ] 金额是否带币种单位和千分位？
- [ ] 确认对话框是否说明了操作后果？
- [ ] 文案是否符合品牌语气（理性/专业/直接）？
