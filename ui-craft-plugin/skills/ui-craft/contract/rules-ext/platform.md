# Platform 扩展规则

> 交易所 + Web3/Pay 共享的扩展规则，继承 `contract/rules.md` 全局规则。

---

## R-P1: 数据精度显示 [B]

```
价格类数据必须遵循精度规范：
✅ 价格显示支持最多 8 位小数（按币种配置）
✅ 数量显示支持最多 4 位小数
✅ 百分比保留 2 位小数
❌ 截断精度导致显示为 0
❌ 无意义的末尾零（如 1.50000000）
```

## R-P2: 实时数据更新感知 [W]

```
涉及行情/余额等实时数据的组件：
✅ 数据变化时有过渡动效（数字滚动/颜色闪烁）
✅ 加载中状态使用骨架屏而非 spinner
❌ 数据刷新时页面跳动
❌ 无加载状态直接展示空白
```

## R-P3: 涨跌色语义 [B]

```
涨跌色使用语义变量，支持地区切换：
✅ var(--color-up) / var(--color-down)
✅ 默认：绿涨红跌（部分地区反转）
❌ 硬编码 green = 涨, red = 跌
```

## R-P4: 信息密度 [W]

```
平台页面信息密度高于活动页：
✅ 紧凑间距（card padding 16-24px vs 活动页 32-48px）
✅ 单行展示多维度数据
✅ 表格行高 40-48px（vs 活动页 56px）
❌ 大面积留白浪费屏幕空间
```

## R-P5: 币种图标规范 [W]

```
币种展示必须使用标准币种图标组件：
✅ 使用 CoinIcon 组件 + 币种代码
✅ 标准尺寸: 16/20/24/32px
❌ 使用文字替代币种图标
❌ 使用非标准尺寸
```
