# 设计契约（元层）

> 设计契约是做所有设计决策时的依据，贯穿所有层级。**所有风格文件（domains/campaign/styles/*.md）必须遵守本契约。**

---

## 不可覆盖规则

R1-R30 共 30 条强制规则，完整定义见 `rules.md`，**任何风格文件都不得覆盖**。

分为 8 类：
- **样式系统**（R1-R2）：CSS 变量 + CSS Modules，禁止硬编码
- **颜色**（R3-R6）：通过 var() 引用，强调色/状态色语义，深浅模式一致
- **间距**（R7-R9）：4px 基准，同类一致，区块层级递增
- **圆角**（R10-R11）：层级关系（弹窗 >= 卡片 >= 按钮 >= 输入框），同类一致
- **排版**（R12-R15）：禁止指定 font-family，标题不跳级，字重分层，行高规范
- **布局**（R16-R18）：最大宽度 1200px，响应式断点，触摸目标 44px
- **图标**（R19-R22）：禁止 emoji，必须 @gate/iconfont
- **组件 + 可访问性**（R23-R27）：CTA 唯一性，对比度，焦点状态，语义化

## 1. 视觉情绪

### 情绪关键词
- **稳定** — 不激进、不冒险、可预测
- **沉静** — 不喧闹、不浮躁、有质感
- **克制** — 装饰适度、留白恰当、不堆砌
- **高端** — 细节精致、比例和谐、有品味

### 情绪反面（避免）
- 浮夸 — 过度装饰、炫技式设计
- 喧闹 — 颜色过多、元素拥挤
- 廉价感 — 比例失调、细节粗糙
- 不稳定 — 风格混乱、缺乏一致性

## 2. 品牌调性

### 人格特征
- **可信赖** — 不夸大、不承诺收益、不隐藏风险
- **专业** — 结构清晰、行为可预测、层次分明
- **理性** — 数据驱动、结论先行、不做情感操控
- **全球化** — 中性语言、无文化偏见、国际化优先

### 语言风格
- 直接而非含蓄
- 数据驱动而非情感导向
- 简洁而非冗长

## 3. 决策原则

当面临设计选择时的判断依据：

| 选择 | 优先 |
|------|------|
| 简洁 vs 丰富 | 简洁 |
| 对称 vs 不对称 | 对称 |
| 强调 vs 克制 | 克制 |
| 装饰 vs 功能 | 功能 |
| 留白 vs 填充 | 留白 |
| 对比 vs 和谐 | 和谐 |

### 特殊情况
- **CTA 按钮** — 优先强调，但不过度
- **关键数据** — 优先强调（金额、收益率等）
- **警告信息** — 优先对比（使用状态色）

## 4. "精美"标准

定义什么算高质量的设计输出：

### 层次分明 → R13/R14
- 主次清晰，视觉流向明确
- 重要信息一眼可见
- 次要信息不干扰主信息

### 比例和谐 → R7/R8/R9/R10
- 间距有节奏（遵循 4px 基准）
- 尺寸有逻辑（遵循组件 Token）
- 符合 8px 基准网格

### 细节一致 → R11/R14
- 圆角统一（同类组件相同圆角）
- 阴影统一（同类元素相同阴影）
- 字重统一（相同样式使用相同字重）

### 留白恰当 → R9
- 不拥挤（元素之间有呼吸空间）
- 不空洞（无意义的空白）
- 有节奏（间距有变化但有序）

## 5. 风格覆盖

当使用域的风格定义时（Campaign: `styles/*.md`，Exchange/Web3: `domain.md` 品牌色覆盖），风格定义会覆盖本契约的对应部分。

### 覆盖规则
- 风格/域定义中的 CSS 变量值优先级高于本契约默认值
- 未定义的变量使用域 semantic.md 的默认值
- 冲突时，风格/域定义的"决策原则"优先

### 不可覆盖
- **rules.md 中定义的所有规则**（R1-R30）
- 风格/域定义只能定义 CSS 变量值和组件变体，不能修改规则本身

## 6. 设计任务双层覆盖

设计规范类任务（如「设计空状态」「设计卡片组件」）的输出必须同时覆盖 UX 层和实现层：

| 层面 | 必须引用 |
|------|---------|
| UX | R19/R20 图标、R23 CTA、R30 文案 |
| 实现 | R1 CSS 变量、R2 CSS Modules、R3 禁止硬编码 |

## 7. 代码输出前强制检查

> 执行 `verification-protocol.md` 定义的 3-Pass 验证管线（Pass 1 静态扫描 -> Pass 2 违规报告 -> Pass 3 修复循环）。
> 规则的 severity/check/fix 定义见 `rules.md`。`[B]` 违规阻断输出，`[W]` 违规标注提醒。
