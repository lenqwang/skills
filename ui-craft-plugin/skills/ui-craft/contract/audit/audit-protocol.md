# 设计评审协议（Audit Protocol）

> 从"合规检查"（R1-R30）升级到"设计质量评审"。
> R1-R30 检查"对不对"，本协议检查"好不好"。
> 改编自 Impeccable audit/critique skills，适配 Gate.io 品牌体系。

---

## 定位

```
contract/verification-protocol.md   →   规则合规（自动扫描，输出前必须执行）
audit-protocol.md          →   设计质量（可选执行，用于提升输出品质）
```

**触发时机：**
- 用户要求评审设计质量时
- 校准循环（Phase 4）中 L1-L4 通过后可选执行
- 复杂页面（>3 个活动区块）生成完成后建议执行

---

## 评审维度（7 项）

### 1. 视觉层次

> 信息的主次是否一眼可分？视觉流向是否自然？

| 检查点 | 评分标准 |
|--------|---------|
| 3 秒测试 | 闭眼 3 秒后重新看页面，第一眼看到的是否是最重要的信息（Hero 标题/主 CTA）？ |
| 视觉流向 | 从上到下的阅读路径是否连贯？是否有元素打断自然流向？ |
| 信息分组 | 相关信息是否通过间距/容器/颜色明确分组？不相关信息是否有足够分隔？ |
| 主次对比 | 主标题与正文的字号/字重差异是否足够（≥1.5 倍字号差）？ |

**评分：** `强` / `中` / `弱`

---

### 2. 反 AI 模式化（R29 深度评审）

> 设计是否有辨识度？还是看起来像"任何 AI 都能生成的"？

| 检查点 | 评分标准 |
|--------|---------|
| 配色辨识度 | 是否能通过配色判断这是 Gate.io 的页面（品牌蓝/强调绿/暗色背景）？ |
| 布局独特性 | 页面结构是否有针对内容的定制，而非通用卡片网格堆叠？ |
| 细节意图 | 渐变/阴影/圆角的使用是否有明确的信息目的（强调/层级/分组）？ |
| 排版品质 | 字号/字重的变化是否创造了节奏感，而非全页统一？ |

**评分：** `强` / `中` / `弱`

**弱的信号：** 换掉 logo 后看不出是哪个品牌的页面。

---

### 3. 空间节奏

> 间距不只是"符合 4px 基准"，更要有节奏感。

| 检查点 | 评分标准 |
|--------|---------|
| 层级递增 | 元素间距 < 区块间距 < 页面间距，是否形成清晰的三级递增？ |
| 留白均衡 | 是否有"局部拥挤 + 大片空白"的失衡？ |
| 呼吸感 | Hero 区是否有足够留白让核心信息"呼吸"？ |
| 紧凑 vs 宽松 | 信息密度是否匹配场景（活动页适度紧凑，产品页适度宽松）？ |

**评分：** `强` / `中` / `弱`

---

### 4. 色彩平衡

> 颜色使用是否克制且有意图？

| 检查点 | 评分标准 |
|--------|---------|
| 60-30-10 法则 | 主色（背景/基础色）~60%，辅助色~30%，强调色~10%？ |
| 强调色克制 | `var(--accent)` 使用点 ≤ 3 处？品牌蓝是否只用于操作元素？ |
| 对比度舒适 | 文字可读性好，且不会因对比度过高而刺眼？ |
| 色彩一致性 | 同类元素是否使用同一色彩角色（不混用 brand 和 accent）？ |

**评分：** `强` / `中` / `弱`

---

### 5. 动效意图

> 每个动画是否有明确的交互目的？

| 检查点 | 评分标准 |
|--------|---------|
| 目的性 | 每个 transition/animation 能否说出它的目的（反馈/引导/因果）？ |
| 节奏分层 | 不同类型的动效是否有不同的时长（反馈快/布局慢）？ |
| 性能安全 | 是否仅 animate transform/opacity？无 layout property 动画？ |
| 减弱动效 | 是否包含 `prefers-reduced-motion` 处理？ |

**评分：** `强` / `中` / `弱`

---

### 6. 文案质量（R30 深度评审）

> 用户能否在不思考的情况下完成操作？

| 检查点 | 评分标准 |
|--------|---------|
| CTA 清晰度 | 看到按钮能否立刻知道点击后会发生什么？ |
| 错误可恢复 | 错误信息是否告诉用户怎么解决，而非只说"失败了"？ |
| 空状态引导 | 空状态是死胡同还是引导入口？ |
| 术语一致 | 同一概念是否全页使用同一术语（不混用"奖励/奖品/奖金"）？ |

**评分：** `强` / `中` / `弱`

---

### 7. 品牌调性匹配

> 设计是否传达了 contract/design-contract.md 定义的情绪？

| 检查点 | 评分标准 |
|--------|---------|
| 稳定感 | 页面是否给人"可靠、不冒险"的感觉？ |
| 沉静感 | 视觉是否安静而有质感，而非喧闹拥挤？ |
| 克制感 | 装饰是否适度？有没有过度的渐变/光效/动画？ |
| 高端感 | 细节是否精致（间距均匀、对齐精确、比例和谐）？ |

**评分：** `强` / `中` / `弱`

## 评审输出模板

```yaml
audit:
  date: YYYY-MM-DD
  target: "<页面名称>"
  style: "<style 名称>"

  scores:
    visual_hierarchy: 强 / 中 / 弱
    anti_ai_slop: 强 / 中 / 弱
    spatial_rhythm: 强 / 中 / 弱
    color_balance: 强 / 中 / 弱
    motion_intent: 强 / 中 / 弱
    copy_quality: 强 / 中 / 弱
    brand_alignment: 强 / 中 / 弱

  overall: 强 / 中 / 弱    # 取最低分

  issues:
    - dimension: "空间节奏"
      detail: "Hero 区留白不足，标题与倒计时间距仅 16px"
      suggestion: "增加到 32px，给标题更多呼吸空间"
      severity: minor / major

  strengths:
    - "配色高度品牌化，品牌蓝 + 强调绿的使用非常克制"
    - "CTA 按钮文案具体明确"

  recommended_actions:
    - skill: "engine/reference/motion-principles.md"
      reason: "动效缺少时长分层，建议参考"
    - skill: "engine/reference/ux-writing.md"
      reason: "空状态文案可优化"
```

## 评审与修复的关系

| 评审结果 | 建议动作 |
|----------|---------|
| 所有维度 `强` | 无需额外操作 |
| 有 `中` | 参考对应 reference 文档优化，非阻断 |
| 有 `弱` | 强烈建议修复后再输出；列出具体 issue 和修复方向 |
| ≥3 项 `弱` | 建议重新评估 style 选择或设计方向 |

---

## 与验证管线的区别

| 维度 | verification-protocol | audit-protocol |
|------|----------------------|---------------|
| 性质 | 规则合规（客观） | 设计质量（主观+客观） |
| 执行时机 | 输出前强制 | 可选，建议执行 |
| 输出 | PASS/FAIL + 违规列表 | 强/中/弱 + 改进建议 |
| 阻断性 | [B] 阻断输出 | 不阻断，但 `弱` 强烈建议修复 |
| 关注点 | 代码层面（CSS 变量/间距/圆角等） | 设计层面（层次/节奏/品牌/文案） |
